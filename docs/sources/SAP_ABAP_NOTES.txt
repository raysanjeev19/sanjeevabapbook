What is Exception Handling?
In SAP ABAP, exception handling means catching and handling errors so the program doesn’t crash, and we can give meaningful messages to the user.
Two Main Types
1. Classical Exception Handling (Procedural)
	•	Mostly used with Function Modules.
	•	Handled using the EXCEPTIONS keyword.
Example:
abap
CopyEdit
CALL FUNCTION 'BAPI_COMPANYCODE_GETDETAIL'
  EXPORTING
    companycode = '1000'
  IMPORTING
    companycode_detail = ls_detail
  EXCEPTIONS
    companycode_not_found = 1
    OTHERS                  = 2.

IF sy-subrc <> 0.
  WRITE: / 'Company code not found'.
ENDIF.
Here, exceptions are handled using sy-subrc.

2. Modern Exception Handling (OOP Style)
	•	Used in methods or class-based exceptions.
	•	Handled using TRY...CATCH...ENDTRY.
Example:
abap
CopyEdit
TRY.
    CALL METHOD lo_emp->get_details
      EXPORTING
        iv_empid = 'E100'.
  CATCH zcx_emp_not_found INTO DATA(lx_error).
    WRITE: / lx_error->get_text( ).
ENDTRY.
💡 Here, the CATCH block handles specific exceptions.

Q1: How many types of internal tables are there in SAP ABAP?
A: An internal table in SAP ABAP is a temporary table used to store multiple records (rows) in memory during the execution of a program.
"There are three types of internal tables in SAP ABAP: Standard, Sorted, and Hashed. 
Standard tables are like simple arrays, Sorted tables keep data in order and allow binary search, and Hashed tables provide the fastest key-based access."

	•	Standard Internal Table
	•	No key is defined.
	•	Data is stored sequentially.
	•	Use when: reading sequentially or small volume of data.
	•	Sorted Internal Table
	•	Data is stored in sorted order based on the key.
	•	Faster binary search is possible.
	•	Use when: frequent searches are needed.
	•	Hashed Internal Table
	•	Uses a unique key like a hash table.
	•	Fastest for reading with a key, but no sorting.
	•	Use when: unique key-based access is needed.
1. Standard Table: A standard table is an internal table with a linear key. You can access its rows using the table index or a key.( Accepts values and allows duplicate entries)
2. Sorted Table: A sorted table is an internal table with a sorted key. The rows are stored in sorted order based on the key fields. (Accepts values and automatically sorts them based on the key fields. It can have duplicate entries unless specified as unique)
3.  Hashed Table: A hashed table is an internal table with a unique hash key. It allows for fast lookup operations using the key.( Accepts values, but it requires a unique key for each entry)

Q4: What is FOR ALL ENTRIES in SAP ABAP?
A: FOR ALL ENTRIES is a keyword used in SAP ABAP to select data from a database table based on the entries in an internal table.
It allows you to:
- Use an internal table as a dynamic filter for your database query.
- Select data from the database table only for the records that match the entries in the internal table.
Example:     SELECT * FROM mara
                            FOR ALL ENTRIES IN it_mara
                           WHERE matnr = it_mara-matnr.

Q5: What is the difference between FOR ALL ENTRIES and INNER JOIN in SAP ABAP?
A: INNER JOIN: Combines rows from two or more tables based on a related column. It fetches data from multiple tables in a single database query.
 FOR ALL ENTRIES: Uses an internal table as a filter to select data from a database table. It fetches data from a single database table based on the entries in the internal table.
Key differences:
- INNER JOIN combines data from multiple tables, while FOR ALL ENTRIES filters data from a single table.
- INNER JOIN is a database operation, while FOR ALL ENTRIES involves an internal table and a database table.
When to use each:
- Use INNER JOIN when you need to combine data from multiple tables.
- Use FOR ALL ENTRIES when you need to filter data from a single table based on a dynamic set of values.

Q6: How many types of joins are there in SAP ABAP, and what are they?
A: There are 4 types of joins in SAP ABAP:
1. INNER JOIN: Returns only the rows that have a match in both tables.
2. LEFT OUTER JOIN (or LEFT JOIN): Returns all rows from the left table and the matching rows from the right table. If no match, the result is NULL on the right side.
3. RIGHT OUTER JOIN (or RIGHT JOIN): Similar to LEFT JOIN, but returns all rows from the right table and the matching rows from the left table.
4. FULL OUTER JOIN (or FULL JOIN): Returns all rows from both tables, with NULL values in the columns where there are no matches.
In simple terms:
- INNER JOIN: Only matching rows.
- LEFT JOIN: All rows from the left table, and matching rows from the right table.
- RIGHT JOIN: All rows from the right table, and matching rows from the left table.
- FULL JOIN: All rows from both tables, with NULL values where there are no matches.

Q1: How many events are there in Module Pool Programming (Dialog Programming) in SAP ABAP?
A:
There are five main events in Module Pool Programming:
	•	PROCESS BEFORE OUTPUT (PBO) – for screen preparation
	•	PROCESS AFTER INPUT (PAI) – for handling user actions
	•	PROCESS ON VALUE-REQUEST (POV) – for F4 help
	•	PROCESS ON HELP-REQUEST (POH) – for F1 help
	•	PROCESS ON COMMIT – used during updates.

Q3:What is the ADD EXIT command in Module Pool Programming?   
A: In Module Pool Programming, when we use buttons with function codes like EXIT, BACK, or CANCEL, SAP treats them as standard exit commands.
To allow the screen to respond to these commands and properly exit the transaction, we need to write the statement ADD EXIT-COMMAND in the screen's PAI (Process After Input) event.
Q4: What is the use of CHAIN and ENDCHAIN in Module Pool Programming?
A: In Module Pool Programming, we use CHAIN and ENDCHAIN to group multiple screen fields together and validate them at once.

Q5:What is the Table Control Wizard in Module Pool Programming ?
A: In Module Pool Programming, a Table Control is used to display and handle multiple rows of data on a screen, similar to a table.

Q6:What is the Tab Strip Wizard in Module Pool Programming and how do you use it?
 A: A Tab Strip in Module Pool Programming is used to create multiple tabs (like in a folder) within the same screen, allowing us to display different information on each tab without navigating to different screens.

1. What is BDC in SAP ABAP?
BDC (Batch Data Communication) is a technique used to upload data into SAP by simulating user inputs on screens (transactions).

2. What are the different methods of BDC?
✅ 1. Call Transaction ✅ 2. Session Method ✅ 3. Direct Input Method (less common)

3. What is the difference between Call Transaction and Session Method?
Feature

Session Method

Faster
Slower

4. What is the transaction code to record a BDC?
SHDB – It is used to record a transaction flow for BDC.

5. What is the structure used for BDC data?
BDCDATA – A standard structure that holds:
	•	Program name
	•	Screen number
	•	Field name and value

6. What are the key function modules used in BDC?
Function Module

7. How do you handle errors in BDC?
	•	In Call Transaction, use parameters like MODE, UPDATE, MESSAGES INTO.
	•	In Session Method, SAP creates automatic error logs via SM35.

8. What are the modes in CALL TRANSACTION?
Mode
Meaning
'A'

9. Can BDC be used for custom transactions?
Yes, if the custom transaction is screen-based and behaves like a standard SAP transaction.

10. What are the limitations of BDC?
	•	Cannot handle controls like ALV grids, dropdowns.
	•	Slow for large data.
	•	Screen-dependent – if screen layout changes, BDC breaks.
